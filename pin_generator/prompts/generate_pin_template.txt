HOW TO USE THIS PROMPT TO GENERATE NEW PIN TEMPLATES
================================================================================

Each pin template is a single Python file (template_N.py) that generates a
600×1067 Pinterest pin as HTML → screenshot → PNG image.

Steps:
  1. Open a new Claude conversation.
  2. Choose a template number (check existing: template_1 through template_5).
  3. Send ONE prompt per template. Each prompt generates ONE complete file.
  4. Save the file as: pin_generator/generators/template_N.py
  5. Restart the pin API server. The new template auto-appears.

IMPORTANT: Send ONE template per message. Each template must have a UNIQUE
design — different layout, different element arrangement, different visual style.

Existing templates for reference (do NOT duplicate these designs):
  template_1 → Full background + dark overlay band (center) + badge + stars + title
  template_2 → Top image + bottom image + colored banner (center) + subtitle + title + website bar

=== ARCHITECTURE ===

The pin is a 600×1067 pixel HTML page rendered to PNG by Playwright.
Everything uses absolute positioning inside a .pin-container div.

The template file imports shared helpers from _base.py:
    from _base import render_pin, apply_domain_style

Each template exports:
    TEMPLATE_ID = "template_{N}"   (string)
    TEMPLATE_DATA = { ... }        (dict — the full template definition)
    STYLE_SLOTS = { ... }          (dict — maps elements to domain color keys)
    FONT_SLOTS = { ... }           (dict — maps elements to "heading" or "body")
    def run(output_dir=None): ...  (function — calls apply_domain_style + render_pin)

=== ELEMENT TYPES ===

1. "div" — colored rectangle/shape
   Required: type, position {top, left}, width, height, z_index
   Optional: background_color, border, border_radius, padding, opacity

2. "text" — text label (flexbox centered, overflow:hidden applied automatically)
   Required: type, text, position {top, left}, width, height,
             font_family, font_size, font_weight, color, z_index
   Optional: text_align, text_transform, letter_spacing, line_height,
             text_shadow, background, background_color, border, border_radius,
             padding, font_style, opacity
   CRITICAL: overflow:hidden is auto-added by _base.py — text NEVER goes outside its box.

3. "stars" — row of star SVGs
   Required: type, count, position {top, left}, star_size, color, z_index

4. "icon" — SVG icon
   Required: type, icon ("search"), position {top, left}, size, color, z_index

=== IMAGE SLOTS ===

Available image keys (system fills these from article data at runtime):
  "background"    → article main_image (full 600×1067)
  "top_image"     → article main_image (top half)
  "bottom_image"  → article ingredient_image or main_image (bottom half)
  "avatar"        → writer avatar or main_image (circular)

Common layout patterns:
  A) Full background (600×1067) with overlay elements on top
  B) Top image + bottom image with a text band between them
  C) Full background with floating card/box overlay
  D) Diagonal split or angled sections
  E) Side-by-side (image left, text right) — less common for pins

=== STYLE_SLOTS — Domain color mapping ===

Maps element names to CSS properties + domain color keys.
When a domain has custom colors, these override the template defaults.

Available color keys:
  "primary"        → domain primary color
  "secondary"      → domain secondary/accent color
  "background"     → domain background color
  "text_primary"   → domain dark text color
  "text_secondary" → domain muted text color
  "border"         → domain border color
  "on_primary"     → #FFFFFF (white text on primary bg)
  "on_secondary"   → #FFFFFF (white text on secondary bg)
  "on_dark"        → #FFFFFF (white text on dark bg)
  "border_accent"  → "2px solid {primary}"

Example:
```python
STYLE_SLOTS = {
    "overlay_band": {"background_color": "primary"},
    "title": {"color": "on_primary"},
    "subtitle": {"color": "secondary"},
    "badge": {"border": "border_accent"},
    "website": {"color": "on_dark", "background_color": "text_primary"},
    "stars": {"color": "secondary"},
}
```

=== FONT_SLOTS — Domain font mapping ===

Maps text elements to font roles: "heading" or "body".
"heading" → uses domain's heading_family (usually serif)
"body" → uses domain's body_family (usually sans-serif)

Example:
```python
FONT_SLOTS = {
    "title": "heading",
    "subtitle": "heading",
    "badge": "body",
    "website": "body",
}
```

=== TEXT PROMPT FORMAT ===

Each text element (except "website") needs a text_prompt AND a matching
field_prompts entry. The prompt tells the AI how to generate that text.

text_prompt format:
  "Generate content for {field} for recipe {{title}}. You are writing the
  {FIELD_TYPE} for a Pinterest food pin. Format: {format rules}. Output ONLY
  the replacement text for {field}, no quotes. Given the user's recipe title
  {{title}}, return only the {field} text."

field_prompts entry uses the same text but in the top-level dict.

Common field types and their formats:
  - title: "title case, 3–8 words" or "ALL CAPS, 2–4 words"
  - subtitle: "2–4 words, cursive style" or "ALL CAPS tagline, 2–5 words"
  - badge: "2–3 words, title case (e.g. 'Top Recipe', 'Best Ever')"
  - time_badge: "lowercase time phrase, 3–6 words (e.g. 'in 10 minutes')"
  - description: "1 short sentence, 8–15 words"

=== CRITICAL RULES ===

1. TEXT MUST STAY INSIDE THE IMAGE: All text elements have overflow:hidden
   auto-applied. BUT you must still size text boxes correctly:
   - Title: width ~500-560px (leave 20-30px margin each side)
   - Height must fit the font_size × expected lines (e.g. 36px font × 2 lines = ~90px)
   - Use line_height: 1.1–1.3 for tight multi-line text
   - Keep font_size reasonable: 28–50px for titles, 18–36px for subtitles

2. LAYER ORDERING: z_index matters. Images = 1, overlay divs = 5–10, text = 15–25.

3. NO HARDCODED DOMAIN INFO: website element text = "yourdomain.com" (replaced at runtime).

4. UNIQUE DESIGN: Each template MUST look different. Vary:
   - Layout (full bg vs split vs card overlay)
   - Text placement (center vs bottom vs top)
   - Color scheme (dark overlay vs white band vs colored gradient)
   - Typography treatment (serif vs sans vs cursive mix)
   - Decorative elements (stars, badges, icons, borders, rounded corners)

5. PROFESSIONAL QUALITY: These pins appear on Pinterest. They must be:
   - Visually attractive and balanced
   - Easy to read (high contrast text on backgrounds)
   - Well-spaced (not cramped, not too spread out)

6. FILE FORMAT: The complete file structure:

```python
#!/usr/bin/env python3
"""Template {N}: {Short design description}."""
from _base import render_pin, apply_domain_style

TEMPLATE_ID = "template_{N}"

STYLE_SLOTS = { ... }
FONT_SLOTS = { ... }

TEMPLATE_DATA = { ... }

def run(output_dir=None):
    apply_domain_style(TEMPLATE_DATA, STYLE_SLOTS, FONT_SLOTS,
                       TEMPLATE_DATA.get("domain_colors"), TEMPLATE_DATA.get("domain_fonts"))
    render_pin(TEMPLATE_ID, TEMPLATE_DATA, output_dir)

if __name__ == "__main__":
    run()
```
